# daily 每日学习记录

本目录用于按“天”为单位整理 TOPIK 学习内容。

以后每天优先新增一个文件：

```text
YYYY-MM-DD.md
```

例如：

```text
2026-06-07.md
2026-06-08.md
```

---

## 每日文件用途

每日文件只记录当天实际学习、复习和测试结果，不再把所有内容拆成很多零散文件。

推荐结构：

```text
# YYYY-MM-DD TOPIK 学习记录

## 1. 今日目标

## 2. 旧内容复习

## 3. 今日新学内容

## 4. 题目训练

## 5. 错题与暴露问题

## 6. 必须回炉

## 7. 下次复习安排

## 8. 是否需要同步到总表
```

---

## 和其他目录的关系

| 目录 | 用途 |
|---|---|
| `daily/` | 每天的学习记录、当天错题、当天回炉内容 |
| `vocabulary/` | 稳定词表、总词表、专题词表 |
| `grammar/` | 稳定语法表、专题语法整理 |
| `mistakes/` | 真题错题专项整理 |
| `practice/` | 练习模板和训练材料 |
| `plan/` | 学习系统、复习规则、长期计划 |

---

## 记录原则

1. 每天一个文件，避免同一天拆出太多零散文件。
2. 当天临时暴露的问题先放进每日文件。
3. 反复错、TOPIK 高频、影响做题判断的内容，再同步到 `vocabulary/` 或 `grammar/`。
4. 做对一次不代表掌握，只进入观察；连续多次正确后才降频。
5. 旧文件暂不强行迁移，先从新日期开始执行每日制。
